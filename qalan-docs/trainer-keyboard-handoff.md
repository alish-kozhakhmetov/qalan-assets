# Trainer Keyboard — бриф для верстки (Claude Design)

Цель: точно собрать кастомную клавиатуру тренажёра для демо-ролика.

## Что передавать (по приоритету)
1. **`trainer-keyboard.standalone.html`** (рядом с этим файлом) — самодостаточный
   эталон: открывается в браузере, рендерит клавиатуру 1:1 со всеми 5 раскладками,
   переключателем и long-press (раскрытие по сетке + затемнение). Токены DS зашиты.
   Это самый точный артефакт «как должно лечь».
2. **Этот бриф** — то, чего по статичным макетам не видно: поведение, токены, маппинг.
3. **Ссылка на Figma** (раздел 1) — для реальных иконок, переменных и шрифта.

## Три «не» — обязательно проговорить Claude Design
1. Long-press — это **раскрытие клавиши по сетке + затемнение фона**, НЕ всплывающая карточка.
2. Вложенности только где `Nesting=true` (Advanced: `< > √ ( ) ,`; Additional: `sin cos tan ctg`). В Basic/abc/ABC — нет.
3. Иконки (`÷ × − +`, `√`, backspace, edit/chat/shift) и токены/шрифт — тянуть из **Figma**, а не копировать из кода (в standalone иконки — текст/приближения).

## 4. Раскладки (5 вариантов Type) — точные символы

**Basic** (числа, базовый режим)
```
<   >   ≠   =   ⌫
x⁄y 7   8   9   ÷
x²  4   5   6   ×
x³  1   2   3   −
( ) ;   0   ,   +
```

**Advanced** (числа, расширенный)
```
<   >   =   ≠   ⌫
x⁄y 7   8   9   ÷
x²  4   5   6   ×
x³  1   2   3   −
√   ( ) 0   ,   +
```

**Additional** (функции)
```
sin cos tan ctg ⌫
∑   ∑ᵃᵦ ∫   ∫ᵃᵦ ±
∈   ∉   ⊂   ⋃   ⋂
∞   e   c   log lg
∅   !   a⃗   π   ‾
```

**abc** (буквы, нижний регистр)
```
⇧ a b c d e
f g h i j k
l m n o p q
r s t u v w
x y z α β γ
δ θ λ σ υ φ
```

**ABC** — то же, латиница A–Z (верхний регистр), греческие те же, Shift активен.

Примечания:
- Операторы `÷ × − +` и `√` — **иконочные** клавиши (SVG), оранжевый столбец = `÷ × − +`.
- `x⁄y` — дробь (стопкой), `x²`/`x³` — со степенью.

## 5. Три свайп-страницы
Точек 3 — это 3 страницы свайпом:
1. **Числа** (Basic ⇄ Advanced — переключение режима)
2. **Функции** (Additional)
3. **Буквы** (abc ⇄ ABC — переключение Shift)

## 6. Долгое нажатие (вкладыши) — ВАЖНО, по статике не видно
У клавиши есть булево свойство **`Nesting`**. Вкладыш есть ТОЛЬКО где `Nesting=true`:
- **Advanced**: `<`  `>`  `√`  `( )`  `,`
- **Additional**: `sin`  `cos`  `tan`  `ctg`
- **Basic / abc / ABC**: вложенностей нет.
  (`∑` и `∫` в DS помечены `Nesting=true`, но компонента-вкладыша для них пока нет — оставить без попапа или досбора́ть.)

**Механика раскрытия:**
1. По долгому нажатию клавиша **раскрывается в сторону прямо по сетке** — доп.
   символы ложатся ровно в соседние ячейки (та же ширина 65 и gap 4), без сдвига.
2. Ширина вкладыша = **2 или 3 ячейки** (138 / 211 px).
3. Направление: **вправо**, если активный символ слева; **влево**, если активный
   справа (активная клавиша остаётся на своём месте).
4. Активный символ — **синий** (`bg/brand`, белый текст); альтернативы — белые ячейки.
5. Остальная клавиатура **затемняется** скримом (≈ чёрный 45%).

**Содержимое вкладышей** (`*` = активный/по умолчанию):
| Клавиша | Ячейки (слева→направо) | Раскрытие |
|---|---|---|
| `<` | `<*`  `≤` | вправо |
| `>` | `>*`  `≥` | вправо |
| `√` | `√*`  `ⁿ√` | вправо |
| `( )` | `( )*`  `[ ]`  `{ }` | вправо |
| `,` | `.`  `;`  `,*` | влево |
| `sin` | `sin*`  `arcsin` | вправо |
| `cos` | `cos*`  `arccos` | вправо |
| `tan` | `tan*`  `arctan` | вправо |
| `ctg` | `arcctg`  `ctg*` | влево |

## 7. Для демо-ролика (анимация)
Последовательность, которую стоит показать:
1. Появление клавиатуры (Basic), курсор в поле ответа.
2. Набор: тап по числам/операторам (лёгкий press-стейт клавиши).
3. Свайп на страницу функций (Additional), тап `sin`.
4. Долгое нажатие на клавишу с вложенностью (`<` или `sin`) → затемнение +
   раскрытие вкладыша по сетке → выбор альтернативы.
5. Тап «Проверить» (brand-пилюля).
